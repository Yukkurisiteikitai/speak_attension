# 0002: 抜け漏れ検知を会議モードの価値に置く

Date: 2026-07-08  
Status: accepted (superseded by 0004)

## 決定

会議モードでは、トピックグラフそのものではなく、会議後に未決事項を根拠付きで示す抜け漏れレポートを価値の中心に置く。

## 背景

議事録要約は差別化しにくく、会議で未解決の事項を把握できることに価値を見出した。一方で、リアルタイムの不確かな提案は信頼を損なうため、判定品質の測定を優先した。

## 影響

- `meetingReport` と評価フィードバックを実装し、レポートを改善するためのデータを出力する。
- 未検証のギャップ候補は会議中に強い提案として表示しない。
- この決定は会議モードに関するものであり、アイデア出しモードのレイアウトには適用しない。
