# 0009: 会議ライブマップを意味階層ツリーにする

Date: 2026-07-21  
Status: accepted

## 決定

会議ライブマップは、発言をルールベースで「話題・課題・原因・アクション・別案・発言」に分類し、左から右へ親子関係をたどる意味階層として表示する。相槌は除外し、追加済みノードの親は自動で付け替えない。「以上です」「今日はここまでです」などの終了表現はノードにせず現在話題を閉じ、次の有意味な発言を会議ルート直下の新しい話題として追加する。誤判定は利用者が役割と親を修正できる。

ライブ意味階層は既存の`MeetingGraph`と並行して保持する。`MeetingGraph`、Focus、カバレッジ、抜け漏れ分析、会議整理は従来どおり動作させる。

## 背景

従来のライブマップは発言ごとに独立した議題を作りやすく、「課題の原因」「原因に対するアクション」といった会話の論理的なつながりを読めなかった。リアルタイム表示へLLMを入れず、会話中に表示が大きく組み替わらない構造が必要だった。

## 影響

- ライブ処理はローカルの純粋な規則だけを使い、LM Studioは既存の非同期後処理に限定する。
- ノードの高評価は単独操作者向けの0/1とし、親子関係とともに評価JSON version 2へ出力する。
- 複数話題は会議ルート直下へ並列配置する。
- 会話ツリーはメモリだけに保持し、リセットまたは再読み込みで消える。
