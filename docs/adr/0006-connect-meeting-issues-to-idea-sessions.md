# 0006: 会議の課題を出典付きでアイデア出しへ引き継ぐ

Date: 2026-07-21  
Status: accepted

## 決定

会議整理マップの「課題」「未解決」項目をユーザーが選択し、その根拠発言から新しいアイデア出しセッションを開始できるようにする。引継ぎ時は会議 ID、議題、整理項目、発言 ID を出典として保持する。

アイデア出しの選別は採用・保留・却下の3状態とし、グループ名をセッション中に編集できるようにする。3状態と出典は Markdown とセッション JSON の両方へ反映する。

## 背景

会議モードを独立した議事録機能として拡張するだけでなく、会議で見つかった課題を主モードの発想活動へ接続する必要がある。後から判断を検証できるよう、要約だけではなく元発言への参照を失わないことも必要である。

## 影響

- 会議からの引継ぎはユーザーの明示選択でのみ行う。
- 同じ発言が複数の選択項目を支える場合、発言は重複させず複数の出典参照を付ける。
- モード間の受け渡しはローカル状態だけで完結し、LLM や外部サービスを必要としない。
- JSON エクスポートは3状態と会議出典を含む version 2 とする。
