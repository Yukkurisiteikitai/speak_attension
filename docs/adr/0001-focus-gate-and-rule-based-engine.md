# 0001: Focus Gate とルールベースエンジン

Date: 2026-07-08  
Status: accepted (superseded by 0004)

## 決定

会議モードのリアルタイム処理は、説明可能なルールベースのトピック抽出、Intent 判定、Focus Gate を中核にする。Focus は直近の高スコア発話だけで切り替えず、明示的な話題転換と強い直接一致を必要とする。

## 背景

会議中の文脈を保つには、単純なキーワード一致だけでは Focus が不安定になる。固定リプレイケースで再現可能に評価できる、決定的な処理が必要だった。

## 影響

- `src/utils` の会議エンジンは純粋関数と併置テストを維持する。
- 手動 Focus 選択・ロックと、判断根拠を残す Decision Log を採用した。
- 会議モードは副モードとなったが、既存の処理基盤は維持する。
