# 0005: 終了時の会議整理はローカルLLMと規則ベースを併用する

## Context

ライブのトピック抽出は低遅延を優先するため、相槌や短い言い換えも個別トピックになりうる。会議終了時には、課題・懸念・決定・担当・期限を根拠発言とともに読み返せる構造が必要である。

## Decision

「会議を整理」の明示操作で、まず規則ベースの整理マップを生成する。LM Studio が設定済みなら、その結果と全発言を入力にして議題統合と要点化を非同期で行う。LLMの出力は既存発言IDを根拠に持つ要点だけを受理し、失敗時は規則ベース結果を保持する。

整理結果はライブの `MeetingGraph` と分け、タイトル編集も整理結果に限定する。原文は要点の配下で開閉できるようにする。

## Consequences

- LM Studio がなくても終了時マップを利用できる。
- ライブのセグメント処理は規則ベースのままで、LLMは会議後の非同期層に限定される。
- 整理後に発言が増えた場合は結果を自動変更せず、再整理を促す。
