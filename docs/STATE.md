# Current State

最終更新日: 2026-07-17

## 現在できること

アプリはアイデア出しモードを既定で開く。日本語音声または手入力からキーワードを収集し、発言順の放射状マップに表示する。収集後はグループ化して左右のマインドマップに遷移し、採用するキーワードを選択して、出典発話を保った Markdown とセッション JSON を出力できる。

会議ダッシュボードは副モードとして利用できる。トピック抽出、カバレッジ追跡、Focus の手動選択・ロック、リプレイ、会議後の抜け漏れレポートと評価データ出力を備える。開発サーバーは `npm run dev`、アプリは `http://127.0.0.1:5173/`、ローカル WebSocket は `ws://127.0.0.1:8787` を使う。

## 現在のフェーズ

アイデア出しモードの使い勝手と品質を検証する段階。放射状表示とグループ化後のマインドマップは、長短の日本語ラベルでも重ならないレイアウトテストで保護されている。会議モードの基盤は保守するが、現在の主な製品判断はアイデア出しを優先する。

変更の検証は `npm run check` を基本とし、UI またはビルドに影響する変更では `npm run build` も実行する。

## 既知の未解決課題

- 放射状レイアウトの衝突回避は、セッション規模（数十キーワード）には十分だが、数百件規模では空間インデックスが必要になる可能性がある。
- キーワード・会議トピックの抽出はヒューリスティックであり、状態はリロードで失われる。
- 音声入力はブラウザーの Web Speech API に依存する。ローカル WebSocket のログ中継は任意である。
