# Current State

最終更新日: 2026-07-17

## 現在できること

アプリはアイデア出しモードを既定で開く。日本語音声または手入力からキーワードを収集し、発言順の放射状マップに表示する。収集後はグループ化して左右のマインドマップに遷移し、採用するキーワードを選択して、出典発話を保った Markdown とセッション JSON を出力できる。LLM 設定(接続先とモデル)は localStorage に永続化され、アイデア出しモードと会議モードで共有される。

会議ダッシュボードは副モードとして利用できる。トピック抽出、カバレッジ追跡、Focus の手動選択・ロック、リプレイ、会議後の抜け漏れレポートと評価データ出力を備える。会議マップは議題を枝、各枝にひも付く発話を時系列の葉として描画し、議題ごと・全体で会話履歴を開閉できる。「会議を整理」を押すと、規則ベースの整理マップを即時に作り、LM Studio が設定されていれば根拠発言を保った議題統合・要点化で更新する。LM Studio の整理中は、発言数に基づく残り時間の目安と進捗バーを表示する。ローカルLLMの応答は通常800トークン、会議整理は1200トークンに制限し、長すぎる生成で処理が占有されることを防ぐ。整理マップでは要点ごとに原文を開閉でき、議題・要点のタイトルだけをセッション中に修正できる。開発サーバーは `npm run dev`、アプリは `http://127.0.0.1:5173/`、ローカル WebSocket は `ws://127.0.0.1:8787` を使う。

## 現在のフェーズ

アイデア出しモードの使い勝手と品質を検証する段階。放射状表示とグループ化後のマインドマップは、長短の日本語ラベルでも重ならないレイアウトテストで保護されている。会議モードの基盤は保守するが、現在の主な製品判断はアイデア出しを優先する。

変更の検証は `npm run check` を基本とし、UI またはビルドに影響する変更では `npm run build` も実行する。

## 既知の未解決課題

- 放射状レイアウトの衝突回避は、セッション規模（数十キーワード）には十分だが、数百件規模では空間インデックスが必要になる可能性がある。
- キーワード・会議トピックの抽出はヒューリスティックであり、状態はリロードで失われる。
- 音声入力はブラウザーの Web Speech API に依存する。ローカル WebSocket のログ中継は任意である。
