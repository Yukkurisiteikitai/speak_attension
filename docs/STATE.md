# Current State

最終更新日: 2026-07-21

## 現在できること

アプリはアイデア出しモードを既定で開く。日本語音声または手入力からキーワードを収集し、発言順の放射状マップに表示する。収集後はグループ化して、左から右へ「テーマ → グループ → キーワード」と進む一方向の階層マップに遷移し、グループ名を編集して各キーワードを採用・保留・却下に整理できる。Markdown とセッション JSON は3状態と出典発話を保持する。LLM 設定(接続先とモデル)は localStorage に永続化され、アイデア出しモードと会議モードで共有される。マップは自由にパン・ズームでき、常時表示される「全体を表示」で全ノードを画面内へ戻せる。

会議ダッシュボードは副モードとして利用できる。トピック抽出、カバレッジ追跡、Focus の手動選択・ロック、リプレイ、会議後の抜け漏れレポートと評価データ出力を備える。会議ライブマップはルールベースで相槌を除外し、発言を「話題 → 課題 → 原因 → アクション／別案」の右向き意味階層として即時表示する。「以上です」「今日はここまでです」などはノードにせず現在の話題を閉じ、その後の発言を会議ルート直下の新しい話題として開始する。各ノードは0/1の高評価を付けられ、選択後に役割と親を手動修正できる。高評価と親子関係はversion 2の評価JSONへ含まれる。既存の議題・カバレッジ分析はライブツリーと並行して維持される。右レールは「進行」「分析」のタブ、下部は初期状態で閉じた「入力・再生」ドックに整理されている。「会議を整理」を押すと、規則ベースの整理マップを即時に作り、LM Studio が設定されていれば根拠発言を保った議題統合・要点化で更新する。整理マップの「課題」「未解決」項目を選ぶと、その根拠発言と会議・議題・項目・発言 ID の参照を保ったままアイデア出しセッションを開始できる。LM Studio の整理中は、発言数に基づく残り時間の目安と進捗バーを表示する。ローカルLLMの応答は通常800トークン、会議整理は1200トークンに制限し、長すぎる生成で処理が占有されることを防ぐ。整理マップでは要点ごとに原文を開閉でき、議題・要点のタイトルだけをセッション中に修正できる。開発サーバーは `npm run dev`、アプリは `http://127.0.0.1:5173/`、ローカル WebSocket は `ws://127.0.0.1:8787` を使う。

## 現在のフェーズ

アイデア出しモードの使い勝手と品質を検証する段階。放射状表示、グループ化後のマインドマップ、会議ライブマップは、長短の日本語ラベルでも重ならないレイアウトテストで保護されている。会議モードの基盤は保守するが、現在の主な製品判断はアイデア出しを優先する。利用形態はファシリテーター1人の操作と画面共有を前提とし、複数端末の同時編集は対象外とする。

変更の検証は `npm run check` を基本とし、UI またはビルドに影響する変更では `npm run build` も実行する。

## 既知の未解決課題

- 放射状レイアウトの衝突回避は、セッション規模（数十キーワード）には十分だが、数百件規模では空間インデックスが必要になる可能性がある。
- キーワード・会議トピック・ライブ意味階層の抽出はヒューリスティックであり、状態はリロードで失われる。意味階層の誤判定は選択ノードの役割・親編集で修正できる。
- 音声入力はブラウザーの Web Speech API に依存する。ローカル WebSocket のログ中継は任意である。
